# Критерии оценивания проекта (0-5)

## Назначение

Документ для оценки итоговой работы студента по проекту **«Магазин электрозапчастей»** после доработки:

- восстановления отсутствующего модуля заказов;
- исправления ролевой авторизации.

## Как оценивать

Проверяющий выставляет баллы по критериям ниже.  
Максимум: **20 первичных баллов**.

## Критерии (20 баллов)

### 1) Запуск и базовая стабильность — 3 балла

- `1 балл` — проект запускается (`python manage.py runserver`) без критических ошибок;
- `1 балл` — миграции применяются корректно;
- `1 балл` — основные страницы (логин, каталог) открываются без 500 ошибок.

### 2) Восстановление модуля заказов — 7 баллов

- `2 балла` — восстановлены маршруты и views модуля заказов;
- `2 балла` — реализован список заказов;
- `2 балла` — реализован CRUD заказов (create/update/delete);
- `1 балл` — после операций корректные редиректы и сообщения пользователю.

### 3) Ролевая авторизация и доступы — 6 баллов

- `2 балла` — корректно определяются роли `guest`, `client`, `manager`, `admin`;
- `2 балла` — права доступа в views соответствуют ТЗ;
- `1 балл` — закрыт прямой доступ к защищённым URL;
- `1 балл` — UI (меню/кнопки) соответствует правам пользователя.

### 4) Качество решения — 4 балла

- `1 балл` — код читаемый, без дублирования в ключевых местах;
- `1 балл` — логика не ломает существующий каталог товаров;
- `1 балл` — нет «мертвых» ссылок/битых переходов;
- `1 балл` — базовая самопроверка выполнена (ручной чек-лист или тесты).

---

## Перевод в итоговую оценку 0-5

Формула:

`оценка_0_5 = (набранные_баллы / 20) * 5`

Округление до целой оценки:

`итоговая_оценка = round(оценка_0_5)`

### Быстрая таблица соответствия

- `0-3` балла из 20 → **1**
- `4-7` баллов из 20 → **2**
- `8-11` баллов из 20 → **3**
- `12-15` баллов из 20 → **4**
- `16-20` баллов из 20 → **5**
